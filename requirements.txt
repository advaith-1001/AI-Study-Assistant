# ==========================================
# Environment Variables Required
# ==========================================
# DATABASE_URL: postgresql+asyncpg://user:password@host:5432/dbname
# VECTOR_DB_URL: postgresql://user:password@host:5432/dbname
# SECRET_KEY: Strong random string (use: openssl rand -hex 32)
# CSRF_SECRET: Strong random string (use: openssl rand -hex 32)
# OAUTH_SECRET: Strong random string (use: openssl rand -hex 32)
# API_KEY: Google Generative AI API key
# MAIL_USERNAME: Email address for sending emails
# MAIL_PASSWORD: Email app password (use app-specific for Gmail)
# MAIL_FROM: Sender email address
# FRONTEND_URL: Frontend URL for CORS (e.g., https://yourdomain.com)
# FRONTEND_CALLBACK_URL: OAuth callback URL

# ==========================================
# Installation & Deployment
# ==========================================
# 1. Create virtual environment:
#    python -m venv venv
#    source venv/bin/activate  # On Windows: venv\Scripts\activate
#
# 2. Install dependencies:
#    pip install -r requirements.txt
#
# 3. Set up environment variables:
#    cp .env.example .env
#    # Edit .env with your values
#
# 4. Run database migrations:
#    alembic upgrade head
#
# 5. Start development server:
#    uvicorn main:app --reload
#
# 6. Start production server:
#    uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
#
# ==========================================
# Production Deployment Checklist
# ==========================================
# [ ] PostgreSQL 14+ with pgvector extension installed
# [ ] All environment variables set securely
# [ ] SECRET_KEY is strong and unique
# [ ] cookie_secure=True in core/auth.py (for HTTPS)
# [ ] HTTPS/SSL certificates configured
# [ ] CORS allowed_origins configured for your domain
# [ ] Database backups configured
# [ ] Logging and monitoring set up
# [ ] Rate limiting configured (slowapi)
# [ ] Email service credentials configured
# [ ] Google Generative AI quota verified
# [ ] Database connection pooling optimized
# [ ] Static files and assets served
# [ ] Health check endpoint configured
# [ ] Error tracking (Sentry) configured (optional)
